一、基础语法规范
✅ 允许的语法：
- 节点类型：
  `[文本]`       # 矩形节点
  `{条件}`      # 菱形决策节点
  `((圆形))`    # 圆形节点（较少使用）
- 连接线：
  `-->`         # 实线箭头
  `-.->`        # 虚线箭头
  `-- 标签 -->` # 带简单文字的箭头
🚫 禁止的语法：
- 节点内特殊符号：`()?:""'`\/[]`
- 多行节点文字
- 箭头标签超过3个单词
二、文字格式规则
✅ 标准格式：
- 节点：`[纯文本]` 或 `[驼峰命名]`
- 箭头标签：`-->|单单词|` 或 `-- 标签 -->`
🚫 典型错误示例：
- `[hasNext()]` → 应改为 `[hasNext]`
- `{是否冲突?}` → 应改为 `{冲突检测}`
- `--|复杂 标签|-->` → 应改为 `--|标签|-->`
三、结构最佳实践
• 单个图表不超过8个节点
• 最大嵌套层级不超过5层
• 决策流程必须使用菱形 `{}` 节点
• 必须明确标注开始/结束节点
• 注意带有括号的用""包围 如：B[执行步骤T(n)] 应改为 B["执行步骤T(n)"]
